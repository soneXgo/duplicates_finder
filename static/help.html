Img Duplicates Finder

Search options

You can choose the search method.

  • Recursive search – search in the source folder and subfolders.
  • Search in the current folder – search for duplicates only in the original folder.

You can specify the path of the file, then the search for duplicates will be only for it.

File formats

You can specify the extension of the files you are looking for.

There are 18 different file formats available.
.png, .jpg, .jpeg, .bmp, .dds, .dib, .eps, .gif, .icns, .ico, .pcx, .ppm, .psd, .tga, .tif, .tiff, .webp, .wmf

Image properties

Limitations

You can limit the size of the file, the time it was created and modified.

Сomparison parameters

Algorithm

Algorithms of different speeds and accuracy are available.

ORB
ORB is the most accurate algorithm, but the slowest. It finds copies of different extensions and is resistant to turns, noise, scaling, cropping, and lighting changes.
Recommendation: use on a small dataset.
Learn more about ORB
aHash
aHash is fast, but less resistant to distortion than pHash.
Learn more about aHash
bHash
bHash is resistant to local changes in the image, but less resistant to global transformations.
Learn more about bHash
dHash
dHash is resistant to brightness changes, but sensitive to image rotation.
Learn more about dHash
mHash
mHash is more resistant to changes in brightness and contrast than aHash, and is also more resistant to noise and distortion than pHash.
Learn more about mHash
pHash (set by default)
pHash is the most optimal in terms of accuracy and speed.
Recommendation: use these algorithms to find similar images faster and more accurately.
Learn more about pHash
MD5, SHA-1 (160-bit), SHA-2 (256-bit), SHA-2 (384-bit), SHA-2 (512-bit)
MD5, SHA-1 (160-bit), SHA-2 (256-bit), SHA-2 (384-bit), SHA-2 (512-bit) algorithms are arranged in ascending order of computational cost and accuracy.
Recommendation: use on a large amount of data to quickly search for absolutely identical images.
Learn more

Similarity

You can specify the percentage of similarity of the images, where 100% are identical images. It is not recommended to use values less than 50%.


Comparison size

The size to which the original image is reduced. The smaller the size, the faster the algorithm works, but the less accurate the search results become.


Quick Search

If this option is enabled, a faster, but less accurate bHash algorithm will be used.

Max duplicates

You can specify the maximum number of duplicates found after which the search will stop.

Same properties

You can specify which identical properties the original image and duplicates should have.

Modifications

You can specify the properties that a duplicate may have compared to the original image.
Warning! This slows down the algorithm.
